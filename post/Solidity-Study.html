智能合约数据类型

Solidity是一种用于编写智能合约的编程语言，它支持多种数据类型，以下是详细介绍：

值类型（Value Types）

  • 布尔类型（bool）： - 取值只有 true（真）和 false（假）两种情况，常用于条件判断，例如在 if 语句中判断某个条件是否满足。示例：bool isDone = true;，可以在函数中根据业务逻辑来设置其值，并基于它进行后续的流程控制。

  • 整数类型（int/uint）： - int 表示有符号整数，即可以表示正数、负数和零。比如 int8，int16，int24 等，其中数字表示所占的位数，位数越大，表示的数值范围越广，例如 int8 的取值范围是 [-128, 127]。 - uint 则是无符号整数，只能表示零和正数，像 uint8 的取值范围是 [0, 255]。示例：uint256 myVariable = 100;，常用于表示数量、计数器、索引等场景，比如记录某个列表中的元素个数。

  • 固定大小字节数组（bytes1，bytes2，…，bytes32）： - 用于存储固定长度的字节数据。例如 bytes32 可以存储长度为 32 字节的数据，常用于存储哈希值等，像在以太坊中，合约地址、交易哈希等很多数据结构的存储都会用到 bytes32。示例：bytes32 hashValue = 0x1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef;

  • 可变大小字节数组（bytes 和 string）： - bytes 用于存储字节数据序列，它的长度是可变的，可以动态地添加或删除字节元素。比如可以用来存储用户上传的文件字节流（当然实际应用中可能还涉及复杂的存储方案配合）。 - string 类型用于存储字符串，本质上也是以字节形式存储，不过它主要用于文本相关的数据处理，例如存储用户的姓名、描述信息等。示例：string myString = "Hello, World!"; bytes myBytes = new bytes(5);

  • 枚举类型（enum）： - 用于定义一组具名的常量值，方便在代码中清晰地表示特定的状态或选项。例如定义一个表示订单状态的枚举：

    这样就可以通过 myOrderStatus 变量来方便地表示和操作订单处于何种状态了。

  • 地址类型（address）： - 用于表示以太坊的账户地址，既可以是外部账户（由用户私钥控制的普通钱包地址），也可以是合约账户地址。它有一些相关的成员函数，比如可以使用 address.transfer() 来向指定地址发送以太币（有一定的 gas 限制等要求）。示例：address myAddress = 0x1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef;

引用类型（Reference Types）

数组（array）
  • 固定长度数组：在声明时就确定了元素个数，例如 uint[5] fixedArray; 就定义了一个能存储 5 个 uint 类型元素的固定长度数组，元素可以通过索引（从 0 开始）进行访问，像 fixedArray[0] = 10; 就是给数组的第一个元素赋值。
  • 可变长度数组：长度可以动态变化，声明方式如 uint[] dynamicArray;，可以使用 dynamicArray.push() 方法来向数组末尾添加元素，例如 dynamicArray.push(20); 就往数组里添加了一个值为 20 的元素。

内置对象/内建对象

内建对象可以分为两个，分别为block和msg；

block：区块信息
block.coinbase (address): 当前块的矿工的地址
block.difficulty (uint):当前块的难度系数
block.gaslimit (uint):当前块gas的上限
block.number (uint):当前块编号
block.blockhash (function(uint) returns (bytes32)):函数，返回指定块的哈希值，已经被内建函数blockhash所代替
block.timestamp (uint):当前块的时间戳
msg：调用时数据
msg.data (bytes):完整的calldata
msg.gas (uint):剩余的gas量
msg.sender (address):消息的发送方(调用者)（最为重要）
msg.sig (bytes4):calldata的前四个字节(即函数标识符)
msg.value (uint):所发送的消息中wei的数量，联盟链中无需使用此数据

以太坊的四个单位：

wei < gwei < finney < ether

换算关系：

ether = 1000 finney

finney = 1000,000 gwei

gwei = 1000,000,000 wei

tx:交易信息

tx.gasprice(uint):交易的gas价格

tx.origin(address):交易发送方（最原始调用者）。发起交易的外部账户地址（也就是最初触发这个交易链的那个外部地址）
