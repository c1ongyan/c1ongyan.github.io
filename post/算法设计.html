1.1 稳定匹配

稳定匹配问题 Stable Matching Problem

以稳定婚姻匹配为例：当前有N位男生和N位女生最后要组成稳定的婚姻家庭，过程开始之前男生和女生在各自的心目中都按照喜爱程度对N位异性有了各自的排序，男生和女生结婚后，对于每一对男生女生，不会出现比起当前匹配的伴侣互相更喜爱的一对男生女生，即可认为婚姻是稳定的。

稳定：匹配好后，不会有不稳定因素

不稳定因素：两个互相更喜欢的人分别跟别人凑成匹配

G-S算法：

算法特质：

(1)在算法执行过程中，与一个女生结成情侣的男生在她的好感排序里是逐步上升的。

(2)在算法执行过程中，与一个男生结成情侣的女生在他的好感排序里是逐步下降的。

(3)循环至多执行 $n^2$轮就会停止。

### 稳定匹配问题 1. 目标：给定医院和医学院学生之间的偏好关系，设计一个自我强化的录取流程。 2. 不稳定对：如果申请人(x)比起其被分配的医院更偏好医院(y)，且医院(y)比起其录取的某学生更偏好(x)，那么申请人(x)和医院(y)是不稳定的。 3. 稳定分配：不存在不稳定对的分配。这是一种自然且理想的条件，因为个人的自身利益会阻止任何申请人/医院进行私下交易。 ### 稳定匹配问题（男女匹配） 1. 目标：给定(n)个男性和(n)个女性，找到一个“合适”的匹配。参与者对异性成员进行评级，每个男性按照从最喜欢到最不喜欢的顺序列出女性，每个女性也按照从最喜欢到最不喜欢的顺序列出男性。例如以下偏好列表： | | 最喜欢↓ | | 最不喜欢↓ | | —- | —- | —- | —- | | | 1st | 2nd | 3rd | | Xavier | Amy | Bertha | Clare | | Yancey | Bertha | Amy | Clare | | Zeus | Amy | Bertha | Clare | | | 最喜欢→ | | 最不喜欢→ | | —- | —- | —- | —- | | | 1st | 2nd | 3rd | | Amy | Yancey | Xavier | Zeus | | Bertha | Xavier | Yancey | Zeus | | Clare | Xavier | Yancey | Zeus | 分别为男性和女性的偏好概况。 2. 完美匹配：每个人都一一匹配。即每个男性恰好与一个女性匹配，每个女性也恰好与一个男性匹配。 3. 稳定性：在匹配(M)中，如果男性(m)和女性(w)比起他们当前的伴侣更偏好彼此，那么未匹配的对(m - w)是不稳定的，因为这样的不稳定对(m - w)可以通过私奔来改善自身情况。 4. 稳定匹配：不存在不稳定对的完美匹配。稳定匹配问题就是给定(n)个男性和(n)个女性的偏好列表，找到一个稳定匹配（如果存在的话）。 ### 稳定匹配问题示例 - 对于分配(X - C)，(Y - B)，(Z - A)是否稳定的问题，答案是否定的，因为 Bertha 和 Xavier 会倾向于结合在一起。 - 而对于分配(X - A)，(Y - B)，(Z - C)，答案是肯定的，它是一个稳定匹配。 ### 稳定室友问题 - 问题是稳定匹配是否总是存在，答案并非先验地明显。在这个问题中，有(2n)个人，每个人对其他人从(1)到(2n - 1)进行排名，要分配室友对使得不存在不稳定对。例如给定的偏好： | | 1st | 2nd | 3rd | | —- | —- | —- | —- | | Adam | B | C | D | | Bob | C | A | D | | Chris | A | B | D | | Doofus | A | B | C | 可以发现(A - B)，(C - D)会导致(B - C)不稳定；(A - C)，(B - D)会导致(A - B)不稳定；(A - D)，(B - C)会导致(A - C)不稳定。由此可知，稳定室友问题中稳定匹配并不总是存在。 ### 求婚 - 拒绝算法（Gale - Shapley 算法，1962） 1. 算法步骤：首先将每个人初始化为自由状态。然后在还有男性是自由的且没有向所有女性求婚过的情况下，选择这样一个男性(m)，(w)为(m)的列表中他尚未求婚过的第一个女性。如果(w)是自由的，就将(m)和(w)订婚；如果(w)已经有未婚夫(m’)但她更喜欢(m)，则将(m)和(w)订婚，并使(m’)变为自由；否则(w)拒绝(m)。 2. 正确性证明 - 终止性：观察可知男性按照偏好降序向女性求婚，并且一旦女性匹配成功，她就不会再变为未匹配状态，只会“向上交易”。可以证明该算法在最多(n^{2})次循环迭代后终止，因为每次循环一个男人向一个新女人求婚，而总共只有(n^{2})种可能的求婚情况。 3. 正确性证明 - 完美性：通过反证法可以证明所有男性和女性都会被匹配。假设存在某个女性在算法终止时未被匹配，根据前面的观察，这意味着她从未被求婚过，但如果存在未匹配的男性，他最终会向所有女性求婚，所以这种假设不成立。 4. 正确性证明 - 稳定性：同样用反证法证明不存在不稳定对。假设(A - Z)是一个不稳定对，分两种情况讨论：如果(Z)从未向(A)求婚过，那么(Z)更喜欢他在 Gale - Shapley 匹配(S^{})中的伴侣而不是(A)，所以(A - Z)是稳定的；如果(Z)向(A)求婚过，那么(A)拒绝了(Z)，这意味着(A)更喜欢她在(S^{})中的伴侣而不是(Z)，所以(A - Z)也是稳定的。在这两种情况下都得出(A - Z)是稳定的，与假设矛盾，所以不存在不稳定对。 ### 总结 1. 稳定匹配问题：给定(n)个男性和(n)个女性及其偏好，找到一个稳定匹配（如果存在的话）。 2. Gale - Shapley 算法：能够在(O(n^{2}))时间内找到一个稳定匹配。 3. 男性最优性：在男性求婚的版本中，每个男性都会得到他的最佳有效伴侣。这里定义如果存在某个稳定匹配使得男性(m)和女性(w)配对，那么(w)是(m)的有效伴侣。同时也引出问题，男性最优是否以牺牲女性利益为代价。 4. 女性悲观性：实际上，Gale - Shapley 算法找到的是女性最悲观的稳定匹配，即每个女性得到的是她的最差有效伴侣。证明过程与男性最优性类似，假设存在某个女性在匹配中得到的不是最差有效伴侣，那么会推出存在不稳定对，与稳定匹配矛盾。 ### 扩展：住院医师与医院匹配 1. 变体情况 - 一些参与者会将其他参与者列为不可接受。例如居民(A)可能不愿意在克利夫兰工作。 - 男女数量不相等。 - 有限制的一夫多妻制，比如医院(X)可能想雇佣(3)名居民。 2. 不稳定匹配定义：如果存在医院(h)和居民(r)满足以下条件，则匹配(S)是不稳定的：(h)和(r)彼此可接受；并且要么(r)未匹配，要么(r)比起其被分配的医院更偏好(h)；并且要么(h)没有招满所有名额，要么(h)比起其至少一名已分配的居民更偏好(r)。 3. 应用 - 住院医师匹配计划（NRMP）：最初在二战后使用，早于计算机的使用。例如在 3 月的匹配中，有 23000 多名居民参与。存在农村医院困境，即某些农村医院不受欢迎，被许多居民列为不可接受，在 NRMP 匹配中这些农村医院的报名人数不足。而农村医院定理表明，在每个稳定匹配中，农村医院得到的居民是完全相同的。 ### 经验教训 - 课程中学到的重要思想包括分离问题的潜在结构和创建有用且高效的算法，并且这些问题可能具有潜在的深刻社会影响（法律免责声明）。 ### 五个代表性问题 1. 区间调度：输入是具有开始时间和结束时间的一组作业，目标是找到相互兼容（作业不重叠）的最大基数子集。 2. 加权区间调度：输入是具有开始时间、结束时间和权重的一组作业，目标是找到相互兼容的最大权重子集。 3. 二分匹配：输入是二分图，目标是找到最大基数匹配。 4. 独立集：输入是图，目标是找到最大基数独立集（即节点的子集，其中任意两个节点都不通过边相连）。 5. 竞争设施选址：输入是每个节点都有权重的图，游戏中两个竞争玩家交替选择节点，如果一个节点的邻居已被选择，则不能选择该节点，目标是选择最大权重的节点子集。并且提到区间调度有(n\log n)的贪心算法，加权区间调度有(n\log n)的动态规划算法，二分匹配有基于(n^{k})的最大流算法，独立集是 NP - 完全问题，竞争设施选址是 PSPACE - 完全问题。 ### 额外幻灯片内容 1. 再次阐述稳定匹配问题的目标和参与者偏好列表的设定方式，与前面内容类似。 2. 关于解的理解，声称男性最优稳定匹配是弱帕累托最优的，即不存在其他完美匹配（稳定或不稳定）使得每个男性的情况都严格更好。证明过程是假设(A)是在 GS 算法执行过程中最后一个收到求婚的女性，由于算法在最后一个女性收到第一个求婚时终止，所以没有男性会被(A)拒绝，并且与(A)匹配的男性在其他匹配中不会比在男性最优稳定匹配中更好。 3. 关于欺骗行为的讨论，问题是是否存在动机去歪曲自己的偏好概况。假设知道男性的求婚 - 拒绝算法将被运行且知道所有其他参与者的偏好概况，结论是对于男性来说不存在这种动机，但对于一些女性来说是存在的，并且没有机制能够保证稳定匹配且防止欺骗。例如给定的偏好示例中展示了女性可能通过改变偏好来影响匹配结果。 4. 最后总结课程中学到的重要思想，包括问题结构分析和算法创建，以及其社会影响，还幽默地提到历史上男性求婚的传统以及一些关于男女在匹配中的建议，如男性应尽早且频繁求婚、更诚实，女性可以主动约男性等，并且认为理论可以丰富社会生活且有趣，还调侃计算机科学专业的学生能找到最好的伴侣。

Chapter 2 Bssica of Algorithm Analysis 算法设计基础

$O,\Omega,\Theta$

设$T(n)$是一个函数，对应规模为n的输入，某个算法的最坏运行时间。

渐进上界

如果存在常数$c>0,n_0\geq0$,使得对于所有的$n\geq{n_0}$,都有$T(n)\leq{c\cdot{f(n)}}$, 则称T的渐进上界为f,记为$T(n)=O(f(n))$

渐进下界

如果存在常数$\epsilon>0,n_0\geq0$,使得对于所有的$n\geq{n_0}$,都有$T(n)\geq{\epsilon\cdot{f(n)}}$, 则称T的渐进上界为f,记为$T(n)=\Omega(f(n))$

渐进紧界

$T(n)=O(f(n))=\Omega(f(n))$ f(n)既是渐进上界又是渐进下界，则称f为渐进紧界，记为$T(n)=\Theta(f(n))$, 此时T(n)与f(n)增长速度一致，只差一个常数因子。

Chapter 7 Network Flow 网络流

1.流网络问题的定义

增广路径（Augmenting Path）

它用于在Ford-Fulkerson算法中增加网络中从源点（source）到汇点（sink）的流量。以下是关于增广路径的关键点：

  1. 增广路径定义：给定流网络$G = (V, E)$ 和流$ f $，增广路径$p$ 是残差网络 $G_f$ 中从源点 ( s ) 到汇点 ( t ) 的一条简单路径。

  2. 残差网络：残差网络 ($G_f$ ) 是基于当前流$f$ 构建的，它显示了每条边可以增加或减少的流量。

  3. 增加流量：对于增广路径上的每条边 $ (u, v)$，可以增加的流量最大为$c_f(u, v) $，这里$c_f(u, v)$是边 $(u, v)$ 的残差容量。

  4. 残差容量：增广路径 ( p ) 的残差容量$ c_f(p)$ 是路径上所有边的残差容量中的最小值，即$ c_f(p) = \min\{c_f(u, v) : (u, v) \in p\}$。

增广路径的概念是解决最大流问题的基础，通过不断寻找增广路径并沿着这些路径增加流量，可以逐步提高网络中的总流量，直到无法找到更多的增广路径，此时达到最大流。

naïve algorithm: 通过对源图不断寻找增广路径，对每一条路径消耗的最大流量减少对应路径上的容量，为剩余容量，知道无法找到一条增广路径，则算法运行完毕，得到剩余图(Residual Graph),其边的权值对应空闲量

有Flow = Capacity - Residual

具体算法流程：

1.构建一个空闲图，它和容量图对应

2.寻找一条增广路径：

​ a.找出增广路径中的瓶颈容量

​ b.更新空闲图

3.重复2步骤直到无法找到增广路径，算法结束

该算法无法保证找到的流是最大的，会受到路径的寻找顺序的影响。但是这种算法能找到阻塞流。

Chapter 8 NP and Computational Intractability NP与计算的难解性

8.1多项式时间归约

定理：若X能归约成Y,则X≤pY

因为X问题能归约成Y问题解决说明，Y的解决方式能解决X,甚至可能解决比X难的问题，而且X问题至少能用Y的方法将解决，也可能存在更轻便的方法，所以X≤pY （左边的为简单问题，右边为难问题）

定理：

-> X≤pY，若Y能在多项式时间解决，则X能在多项式时间解决。

-> X≤pY, 若X不能再多项式时间解决，则Y不能在多项式时间解决

-> X≤pY Y≤pX , 则X≡pY

定理：顶点覆盖≤p集合覆盖

集合打包问题：给定n个元素的集合U,U的子集的集合S1…,Sm,以及数值k,那么这些集合中是否存在至少k个集合，两两不相交？

构造：与集合覆盖的构造相同，只是取不相交的集合就行。

定理：独立集≤p集合打包

通过小配件归约：可满足性问题

SAT问题和3-SAT问题

SAT问题：给定变量集$X={x_1,x_2,…,x_n}$上的一组子句$C_1,C_2,…,C_k$,是否存在满足的真值赋值？

3-SAT问题：给定变量集$X={x_1,x_2,…,x_n}$上的一组子句$C_1,C_2,…,C_k$,每个长度为3（所有子句恰好包含3个项）是否存在满足的真值赋值？

验证可满足是容易的，但是验证不可满足是困难的，因为要验证所有的可能不满足.

将3-SAT归约到独立集

定理：3-SAT≤p 独立集问题

构造如下：

要素:构建图和k，也弄成需要回答yes/no的提问的格式。

构造一个图G=(V,E),由3k个节点组成，分组为k个三角形，对冲突项添加边。

->三角形构造：将每个子句构成一个三角形，这个时候每个子句中取一个项就是独立集，独立集的个数为子句的个数，从每个子句中取一个就是取这个值为真，子句是析取式，一个为真整个子句就是真的，所有一个子句中只用取一个，对于的子句为三角形，项之间两两互联，为了保证独立也只能取一个，所以这种构造很好的将需求联合起来了。

->三角形之间互联的边：需要注意的是，不同三角形中可能跟有对应的同一个项的情况，为了不能出现矛盾，将项与其对应的否互联起来，保证他们不能同时被取到，形成限制满足了要求。

归约完成：存在一种可满足的情况→存在大小至少为k的独立集

证不可满足比证找到一个可满足的情况要难，需要验证所有的情况

最终观察：归约的传递性

归约具有传递性：如果Z≤pY,且Y≤pX，则Z≤pX

已有：3-SAT≤p独立集≤p顶点覆盖≤p集合覆盖

=> 3-SAT≤p集合覆盖
